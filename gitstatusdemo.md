
# Git Status Demo

This is a sample file for practicing Git commands and understanding how `git status` works in a repository.

## Key Concepts

- **Untracked files**: New files that Git hasn't seen before
- **Staged changes**: Files ready to be committed
- **Unstaged changes**: Modified files not yet staged
- **Committed changes**: Changes already saved in Git history

## Common Commands

Use `git status` to check the current state of your working directory.
